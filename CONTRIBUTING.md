# Contributing

## What to contribute

- Better starter examples for classes/events/todos/categories
- README improvements for onboarding and best practices
- Cross-app compatibility checks (Pro/Web/Desktop)

## Rules

- Follow PlanWell vault schema conventions.
- Keep sample data realistic but simple.
- Prefer additive changes over breaking file structure changes.

## Pull requests

- Explain the problem and why the change helps new users.
- Include updated sample markdown if behavior changes.
